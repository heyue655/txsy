# 太虚书院 · 功能与玩法说明

> 太虚书院是一款**穿越时空的沉浸式读书对话产品**。用户置身于三维星空书海之中，与古代典籍的作者"灵魂"展开跨时代的思想对话，并通过笔谈记录、段位积累，完成一段属于自己的修炼之旅。

---

## 一、整体架构

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 19 + Vite + TypeScript | 移动端 H5 主界面 |
| 3D 渲染 | Three.js + React Three Fiber | 星空书海场景 |
| 后端 | Node.js + Express + Prisma | RESTful API + SSE 流式推送 |
| 数据库 | MySQL | 书籍、对话、用户、配置 |
| AI | 可接入任意 OpenAI 兼容接口 | DeepSeek / Qwen / GPT 等 |

## 二、主界面——星空书海

### 2.1 沉浸式3D场景
- **静态繁星**：5万+ 颗冷色调背景星，带真实透视衰减，5% 过曝光晕显著（可在 `config.js` 调整数量）。
- **闪烁繁星**：50 颗前景闪烁星，前 20 颗黑白彻底闪灭，后 30 颗冷色调彩色规律闪烁。
- **流星**：带 48 节拖尾点阵的粒子流星，随机从左上/右上方斜向穿越，☆ emoji 跟随头部，飞出屏幕后随机冷却重新出现。
- **书籍漂浮**：最多同时显示 15 本书（`VISIBLE_SLOTS`），每本以随机速度从远及近飞过，带动态缩放（由小到大 1×→6× 震撼放大），飞出近端自动回收，换下一本（全书打乱不重复轮播完后再重排）。

### 2.2 视角控制
- **触摸**：单指拖拽转动视角（惯性滑动），双指捏合缩放 FOV（30°～110°）。
- **桌面端**：鼠标拖拽转动，滚轮缩放。
- 松手后带速度衰减惯性，不自动回弹。

### 2.3 书籍标签展示
每本飘过的书显示：书籍 emoji + 书名 + 作者名，随相机朝向自动面向镜头，透视距离越近字越大，颜色在聚焦/暂停状态下切换为该书专属灵魂色。

## 四、书籍发现与导航

### 4.1 分类筛选
支持多标签分类，点击切换（再次点击取消）：
- **古籍** / **文学** / **政治** / **军事** / **管理** / **成长** / **其他**

筛选后，漂浮书海只显示对应分类书籍，「先贤」Tab 列表同步过滤。

### 4.2 搜索
顶部搜索框支持按**书名、作者名、年代**模糊搜索，最多显示 6 条下拉结果，选择后直接聚焦该书并弹出进入对话的按钮。

### 4.3 先贤列表（Tab 切换）
以卡片形式展示所有书籍（书名、作者、年代、分类标签），点击直接进入该书的灵魂召唤流程。

## 五、灵魂对话核心玩法

### 5.1 召唤流程
1. **点击漂浮书籍** → 书籍暂停，弹出书名/作者信息浮窗，显示"进入灵魂对话"按钮；
2. **点击按钮** → 书籍聚焦飞向屏幕中央（FOV 从 75° 缩至 42°），全屏覆盖**灵魂召唤动画**；
3. **召唤动画**：粒子从随机位置向人形轮廓汇聚（~2 秒），18 条能量螺旋线、120 颗环绕粒子、中心光晕脉冲，颜色为该书专属灵魂色；
4. **2秒后自动进入** → 淡入显示 **SoulDialog 对话界面**。

### 5.2 对话界面（SoulDialog）
- **作者开场白**：系统随机选取档案中配置的多条开场白之一（按换行分隔）。
- **流式输出**：AI 回复通过 SSE（Server-Sent Events）实时逐字推送，有打字光标效果。
- **对话历史**：登录用户可连续会话（自动恢复最近一次 `conversationId` 的历史消息）；访客每次重新开始。
- **新对话 / 重新解读**：点击按钮可开始全新会话（有二次确认弹窗防误触）。
- **作者灵魂档案卡**：可展开/收起，显示作者身份、性格、核心观点。

### 5.3 @ 提及其他作者（跨书对话）
- 在输入框中输入 `@` 触发书目选择器（支持按书名/作者名检索）。
- 选中后，系统以目标作者的灵魂档案作为 system prompt，让 AI 扮演被@作者的视角进行回复，并在消息气泡上显示**书名 + 作者名标签**（蓝色区分）。
- 下一条正常消息自动恢复原书作者。

### 5.4 院长对话（DeanDialog）
- 点击 UI 右下角"召唤院长"按钮，触发**院长降临动画**（~0.9秒）后进入对话。
- 院长角色为"太虚院长"，可引导探索书目、推荐书籍、讨论任何学问话题。
- 院长支持 `@` 引用功能：输入 `@` 可引用已与某本书的历史对话片段，院长会结合该上下文作答。
- 院长有独立的会话（每位登录用户独立存档）。

## 六、太虚笔谈（对话笔记）

### 6.1 生成笔谈
- 对话达到一定长度后，底部出现"生成太虚笔谈"按钮；已生成且无新消息时按钮转为"已归档"状态。
- 点击后调用 AI，自动对本次对话进行**总结、提炼精华发言、给出 1-5 分评分**。
- 笔谈内容存入数据库（`note_entries` 表），并记录精彩发言摘录（`highlights` JSON）。

### 6.2 查看笔谈（笔谈 Tab）
- 按时间倒序列出所有笔谈记录，显示书名、得分星级、摘要内容。
- 点击可展开查看完整笔谈正文。

### 6.3 生成分享图
- 每条笔谈可一键生成分享长图（Canvas 绘制）：
  - 深蓝星空背景 + 随机星点；
  - 顶部书名、作者名、评分，品牌水印；
  - 精彩发言以卡片形式展示，读者/作者各有专属强调色和左侧竖线；
  - 底部显示用户昵称、灵魂段位图标、邀请二维码；
  - 直接浏览器打开可长按保存分享。

## 七、灵魂段位系统

用户所有笔谈得分累加为**总积分**，对应以下 8 个段位：

| 段位 | 积分范围 | 称号 | 心印 |
|------|----------|------|------|
| 1 | 0 – 20 | 初入书院 | 📖 蒙昧之心 |
| 2 | 21 – 100 | 初窥门径 | 🌿 问道之心 |
| 3 | 101 – 500 | 渐入佳境 | 🌱 求索之心 |
| 4 | 501 – 1000 | 融会贯通 | ✨ 明澈之心 |
| 5 | 1001 – 2000 | 博闻强识 | 🔥 慧识之心 |
| 6 | 2001 – 5000 | 通儒达道 | 💫 贯通之心 |
| 7 | 5001 – 10000 | 宗师境界 | 🌟 玄思之心 |
| 8 | 10001+ | 太虚先贤 | ☀️ 无上之心 |

"我的"Tab 显示：当前段位图标、称号、积分、段位进度条，以及下一段位的目标积分。

## 八、用户系统

### 8.1 访客模式
- 无需登录即可进入，对任意书籍发起对话。
- 默认最多发送 **3 条消息**（可后台调整）。
- 达到上限后弹出登录/注册引导，显示"访客畅言次数已达上限"提示。
- 访客使用浏览器指纹（`txbt_fp`）做临时身份，历史消息不持久跨设备。

### 8.2 注册 / 登录
- 用户名 + 密码（bcrypt 哈希存储），可选昵称。
- 登录后 JWT Token 存入 localStorage，刷新页面自动恢复登录态。
- 登录用户无消息上限限制，聊天记录永久保存。

### 8.3 邀请码机制
- 注册成功后自动生成专属 16 位邀请码（`inviteCode`）。
- 分享链接格式：`https://域名/?code=XXXXXX`
- 被邀请人访问时自动记录访客指纹；注册时使用邀请码，写入邀请关系（`invitations` 表）。
- 分享图底部会自动嵌入邀请码对应的 QR Code。

### 8.4 个人档案（我的 Tab）
- 显示昵称（可修改）、用户名、灵魂段位卡片；
- 笔谈数量、总积分统计；
- 邀请码展示与复制；
- 退出登录。
- 「玩法规则」入口（弹窗说明产品玩法）。

## 十、作者灵魂档案（AuthorPersona）

每本书对应一份灵魂档案，用于生成 AI system prompt，包含：

| 字段 | 说明 |
|------|------|
| 身份设定 | 作者创作此书时的历史身份与背景 |
| 专业领域 | 学术/技能专长 |
| 社会地位 | 历史地位与影响力 |
| 性格特征 | 鲜明个性描述 |
| 核心观点 | 书中核心思想（JSON 数组，多条）|
| 逻辑武器 | 惯用的论证/思辨方式 |
| 交流风格 | 与人交流时的习惯特点 |
| 口头禅 | 经典语录（分号分隔）|
| 情绪态度 | 喜好、厌恶与热情所在 |
| 知识边界 | 不应谈及的话题/盲区 |
| 说话风格 | 语言特点/文风 |
| 开场白 | 多条风格各异的开场白（换行分隔，随机选取）|

## 十一、管理后台（Admin）

访问路径：`/admin`

| 模块 | 功能 |
|------|------|
| 数据概览 | 书籍数/对话数/笔谈数/用户数等统计 |
| 书籍管理 | 增删改书籍信息，设置分类、灵魂色、排序、启用/禁用 |
| 灵魂档案 | 手动编辑或一键 AI 自动生成（调用激活的 LLM）|
| 大模型配置 | 支持多套 LLM 配置（provider/endpoint/apiKey/model/temperature/maxTokens），激活唯一，内置连通性测试（显示延迟和模型回复）|
| 院长配置 | 配置太虚院长的身份/灵魂档案 |

**AI 自动生成灵魂档案**：后台一键调用当前激活 LLM，根据书名/作者/年代/简介自动生成完整灵魂档案 JSON，管理员确认后保存。

## 十二、数据存储说明

| 表 | 主要用途 |
|----|----------|
| `books` | 书籍基本信息 |
| `author_personas` | 作者灵魂档案 |
| `llm_configs` | 大模型接入配置 |
| `chat_messages` | 全量聊天记录（sessionId + conversationId 双索引）|
| `note_entries` | 太虚笔谈记录（含精彩发言 highlights）|
| `users` | 注册用户（bcrypt 密码）|
| `invitations` | 邀请关系记录 |
| `app_configs` | 应用级 key-value 配置（如访客消息上限）|
| `dean_config` | 院长人格配置 |

```
         └── 我的 Tab → 查看段位积分 → 分享邀请码 → 邀请好友
```
